# Task 1.3.3: Submission Analysis & Legislative Refinement Framework

## Analysis Methodology
All submissions will be coded using NVivo against the three consultation questions plus emergent themes. Each submission receives dual coding for inter-rater reliability. Quantitative sentiment analysis supplements qualitative thematic coding.

## Coding Taxonomy

| Code Category | Sub-Codes | Example Indicators |
| :--- | :--- | :--- |
| **Q1: Investigative Powers** | Proportionality / Safeguards / Warrant requirement / Scope creep / Insufficient powers | "judicial warrant," "fishing expedition," "too weak," "chilling effect" |
| **Q2: Jurisdictional Boundaries** | Ambiguity / Overlap / Gap / Constitutional concern / Referral mechanism | "grey area," "Fiosrú conflict," "courts admin unclear," "Article 30" |
| **Q3: Whistleblower Protections** | Retaliation risk / Anonymity / Cultural barrier / Dedicated liaison / Beyond 2014 Act | "career suicide," "identified easily," "need specialist," "not enough" |
| **Emergent: Implementation** | Resourcing / Timeline / Staffing / IT feasibility / Training needs | "underfunded," "unrealistic," "legacy systems," "recruitment crisis" |
| **Emergent: Trust/Legitimacy** | Political independence / Transparency / Community representation / Historical scepticism | "ministerial puppet," "publish everything," "no community voice," "another whitewash" |

## Triage Protocol for Legislative Amendments

| Submission Concern | Validity Threshold | Legislative Response | Non-Legislative Response |
| :--- | :--- | :--- | :--- |
| Raises constitutional vulnerability | Any credible legal argument | Amend Head immediately; seek updated AGO advice | N/A |
| Identifies operational impossibility | Supported by ≥3 independent stakeholders | Revise Head or add transitional provision | Implementation guidance |
| Proposes enhanced safeguard | Aligns with human rights standards | Add safeguard clause to relevant Head | Regulatory specification |
| Requests expanded jurisdiction | Consistent with policy intent + constitutionally permissible | Extend Schedule 1 or amend Head 19 | MoU or administrative protocol |
| Raises valid concern but outside scope | Genuine issue but incompatible with CJOC model | No amendment | Refer to relevant body/minister; note in Consultation Report |
| Unsupported assertion / anecdotal only | Not corroborated by evidence or other submissions | No amendment | Acknowledge in report; flag for future monitoring |

## Specific Refinement Triggers (Pre-Identified)

Based on PLS hearings and early engagement, the following amendments are anticipated if validated through consultation:

| Anticipated Concern | Trigger Condition | Pre-Drafted Amendment Option |
| :--- | :--- | :--- |
| Whistleblower protections insufficient | ≥5 justice worker submissions citing specific retaliation fears | New Head 35A: Dedicated CJOC Whistleblower Protection Officer + enhanced penalty for retaliation within justice sector |
| Warrant requirement for sensitive data | DPC or ICCL submission identifying disproportionate intrusion | Amend Head 22(2): Require High Court warrant for seizure of legally privileged or health-related personal data |
| Courts Service admin boundary unclear | Courts Service or Judicial Council submission identifying grey zone | Amend Schedule 1: Explicit enumeration of includable/excludable Courts Service functions |
| Referral SLAs too vague | ≥3 oversight bodies requesting certainty | New Head 25(3): Prescribe maximum 10-working-day acknowledgment + 30-day substantive response for referrals |
| Research access blocked | ≥5 academic submissions citing data unavailability | New Head 17(5): Mandate anonymised research dataset publication protocol |

## Quality Assurance
-   Analysis team includes one external academic observer for methodological validation.
-   Draft analysis reviewed by AGO for legal accuracy before publication.
-   All "Not Accepted" decisions require written justification signed by PWG Chair.
-   Consultation Report includes raw code frequency tables for transparency.
